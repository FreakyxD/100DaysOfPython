API Documentation

GET /random

Returns a random cafe from the database.

Response: JSON object containing cafe details or an error message.

GET /all

Retrieves all cafes in the database, ordered by name.

Response: JSON array of cafes or an error message.

GET /search

Searches for cafes by location.

Query Parameters:

  • loc - Location to search for.

Response: JSON array of cafes or an error message.

POST /add

Adds a new cafe to the database.

Form Data Parameters:

  • name (String) - Name of the cafe.
  • map_url (String) - Google Maps URL of the cafe.
  • img_url (String) - Image URL of the cafe.
  • location (String) - Location of the cafe.
  • seats (String) - Number of seats available at the cafe.
  • has_toilet (Boolean) - Indicates if the cafe has a toilet. Accepts "true" or "false".
  • has_wifi (Boolean) - Indicates if the cafe has WiFi. Accepts "true" or "false".
  • has_sockets (Boolean) - Indicates if the cafe has electrical sockets. Accepts "true" or "false".
  • can_take_calls (Boolean) - Indicates if phone calls can be taken inside the cafe. Accepts "true" or "false".
  • coffee_price (String) - Price of coffee at the cafe.

Response: Success message or an error.

PATCH /update-price/<int:cafe_id>

Updates the coffee price of a specified cafe.

Query Parameters:

  • new_price - New coffee price.

Response: Success message or an error.

DELETE /report-closed/<int:cafe_id>

Deletes a cafe from the database.

Query Parameters:

  • api-key - API key for authentication.

Response: Success message or an error.
